# Tutorial: AI Model Fine-Tuning 🧠

To elevate the AI4ALL-SRE laboratory to its peak potential, you can transform the base LLM (Llama 3 8B) into a specialized **SRE-Kernel**. This is achieved through Parameter-Efficient Fine-Tuning (QLoRA) using proprietary incident datasets.

**Goal:** Validate your GPU and run the specialization pipeline.

---

## 1. Verify ML Environment Requirements

You need a Linux environment with an NVIDIA GPU (RTX 3060 12GB minimum recommended) and the official drivers installed.

Run the AI Environment Doctor:
```bash
cd ai-lab
./doctor.sh
```

**Expected Result:**
The script should output `[PASS]` for GPU Detection, NVML Library, and CUDA Toolkit. If it fails, follow the on-screen instructions to install the NVIDIA Container Toolkit.

---

## 2. Bootstrapping the Conda Environment

The fine-tuning pipeline (`unsloth`) is highly sensitive to Python dependency versions. The doctor script provides the exact `conda` creation commands. If you haven't already, install [Miniconda](https://docs.anaconda.com/free/miniconda/index.html).

Create and activate the environment:
```bash
conda create -n sre-ai-lab python=3.10 -y
conda activate sre-ai-lab
```

Install the strictly pinned dependencies generated by the environment setup script:
```bash
./setup_env.sh
```

---

## 3. Run the Specialization Pipeline

With the environment activated, you can now run the fine-tuning job. This process takes the foundation model and trains it on `dataset_template.jsonl`, teaching it our specific GitOps and Kyverno organizational constraints.

```bash
./specialize-model.sh
```

### What happens now?
1. **Checkpointing**: The script uses the HuggingFace `trl` (Transformer Reinforcement Learning) library to train the model.
2. **Quantization**: Once trained, it exports the model into `.gguf` format (4-bit or 8-bit quantized) so it can run locally on consumer hardware.
3. **Ollama Integration**: The newly specialized `sre-kernel` model is automatically imported into your local Ollama instance.

---

## 4. Verification

To verify the fine-tuned model is responding:
```bash
ollama run sre-kernel "Hey, what is the Kyverno policy for privileged containers?"
```

The model should now respond with highly specific knowledge about the `ai4all-sre` repository.
